1️⃣ Transaction rollback in Spring Boot
❓ Problem
@Transactional
public void transfer() {
    debit();
    credit();
    throw new RuntimeException();
}
Why might this NOT roll back?

🧠 Theory (what’s really happening)
Spring transactions work using AOP proxies.

Key rules:

Only runtime exceptions trigger rollback by default.

Method must be invoked via Spring proxy, not self-invocation.

Exceptions must propagate out (not caught).

Checked exceptions require rollbackFor.

🚨 Root causes
Most common failures in real systems:

❌ Self-invocation
this.transfer(); // bypasses proxy → no transaction
❌ Exception swallowed
try {
    debit();
} catch (Exception e) {
    log.error("ignored");
}
❌ Checked exception
throw new Exception(); // no rollback by default
✅ Correct production solution
@Transactional(rollbackFor = Exception.class)
public void transfer() {
    debit();
    credit();
    if (true) {
        throw new RuntimeException("Force rollback");
    }
}
If self-invocation is required:

@Autowired
private TransferService proxy;

proxy.transfer();

Map<String, Optional<Employee>> result =
    employees.stream()
        .collect(Collectors.groupingBy(
            Employee::getDept,
            Collectors.collectingAndThen(
                Collectors.toList(),
                list -> list.stream()
                    .map(Employee::getSalary)
                    .distinct()
                    .sorted(Comparator.reverseOrder())
                    .skip(1) // second highest salary
                    .findFirst()
                    .flatMap(secondSalary ->
                        list.stream()
                            .filter(e -> e.getSalary().equals(secondSalary))
                            .findFirst()
                    )
            )
        ));

@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(AccountNotFoundException.class)
    public ResponseEntity<ApiError> handleNotFound(AccountNotFoundException ex) {
        return ResponseEntity
            .status(HttpStatus.NOT_FOUND)
            .body(new ApiError("ACCOUNT_NOT_FOUND"));
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ApiError> handleGeneric(Exception ex) {
        return ResponseEntity
            .status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(new ApiError("INTERNAL_ERROR"));
    }
}
